Clients and servers

Computer connected to the Web are called clients and servers.

  • Clients are the typical Web user's Internet-connected devices (for example, your computed connected to Wi-Fi, or your phone connected to your mobile network) and Web-accessing softwareavailable on those devices (usually a web browser like Firefox or Chrome).
  • Servers are computers that store webpages, sites, or apps. When a client device want to access a webpage, a copy of the webpage is downloader from the server onto the client machine to be displayed in the userÂ´s web browser.

So What Actually Happens?

  1. The browser goes to DNS server and finds the real adress of the server that the websites lives on.
  2. The browser sends an HTTP request message to the server asking it to send a copy of the website to the client.
  3. If the server approves the client's request, the server sends the client a "200 OK" message, which means "Permission granted! Here you go!", an then starts seding the website's diles to the browser as series of small chunks.
  4. The browser assembles the small chunks into a complete website and displays it to you!
